## Internal linking strategy (homepage → learn → file funnel)

```
Homepage (/)
  ├─ PopularGuides → /learn/{priority-slug}
  ├─ SEO landings → /form-16-filing, /old-vs-new-regime, /itr-deadline-2026
  └─ CTA → /file or /file/import/documents?source=form16

Learn index (/learn)
  └─ Article → RelatedArticles (same cluster) → BlogCTA → /file

Blog mirror (/blogs/{slug})
  └─ Same content + BlogCTA + glossary footer

Glossary (/glossary/{term})
  └─ Link to hub learn article + /file

File funnel (/file → import → companion)
  └─ Contextual links back to learn guides in product (existing companion copy)
```

Rules:

- Every priority article ends with **Check your ITR with LastMinute** → `/file` or Form 16 import.
- Always mention **final filing on incometax.gov.in**.
- Cross-link **minimum 3** sibling learn URLs per article.
- Avoid orphan articles — add to `PRIORITY_GUIDE_SLUGS` or cluster tags.

## Blog writing rules

1. **Tone:** Conversational — a knowledgeable friend in India, not a CA lecture.
2. **Examples:** Indian salaries (₹8L, ₹12L), metro rent, FD interest, job change — not US/EU scenarios.
3. **Compliance:** No guaranteed refund, no “we file for you”, no ITD partnership. Companion + user submits on gov portal.
4. **Structure:** Include **What you should do** and **Common mistake** sections on priority posts.
5. **CTA:** One primary CTA to `/file` or Form 16 import; secondary link to related learn article.
6. **Metadata:** Title ≤ 60 chars where possible; description 150–160 chars with primary keyword.
7. **Freshness:** Update `publishedAt` on material rewrites; note AY/FY in body (AY 2026-27).
8. **Duplicates:** Single slug in `lib/content/learn-articles*.ts`; blogs mirror learn via `blogs.ts`.
